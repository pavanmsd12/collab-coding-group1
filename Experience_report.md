# Experience Report – Collaborative Coding & GitHub Workflow Simulation

## Roles and Contributions
The project was to be done in a group, but during the implementation only one member was present. So the collaboration was simulated using different Git branches to represent contributors. The Admin created the repository, added the starter code and documentation files, and managed pull requests. Contributor 1 implemented the `add_numbers()` function in the `feature-addition` branch and Contributor 2 implemented the `factorial()` function in the `feature-factorial` branch. Both features were later merged into the main branch.
## Challenges Faced
Initially it was a bit confusing to manage Git commands like switching branches, committing changes, and merging pull requests. Since the collaboration was simulated, extra care was needed to make sure the branches were merged correctly.
## Insights on GitHub Workflows
This activity helped understand how version control works in collaborative development. Branching allows developers to work on separate features without affecting the main code, and pull requests help integrate those changes in an organized way.
## Suggestions for Improvement
Planning tasks clearly before starting development can make the workflow smoother. Writing clear commit messages and maintaining simple documentation can also help make collaboration easier.
